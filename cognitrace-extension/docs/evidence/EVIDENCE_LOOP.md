# Evidence Loop Playbook

Goal: convert real user cases into product improvements and test coverage.

## Evidence Case Types

- False-positive dispute support (student/writer challenged by AI detector)
- Academic submission trace proof (coursework integrity proof)
- Client delivery proof (content agency or freelance handoff)

## Weekly Loop

1. Intake: collect new evidence cases with structured template.
2. Triage: classify severity and reproducibility.
3. Product action: create issue(s) for UI, policy, or algorithm updates.
4. Test action: add fixture/test to prevent regression.
5. Publish: summarize lessons in changelog/roadmap updates.

## Operating Metrics

- cases collected per week
- cases converted to actionable issues
- issues linked to tests
- median time from case intake to fix shipped

## Ownership

- Product owner: reviews case quality and user impact
- Engineering owner: defines implementation and test updates
- Community owner: closes the loop with user feedback response
