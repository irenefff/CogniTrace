# Suggested Public Repository Structure

Use this structure when creating the open-source GitHub repository.

## Root

- `README.md`
- `LICENSE`
- `CONTRIBUTING.md`
- `CODE_OF_CONDUCT.md`
- `SECURITY.md`
- `.github/`
- `cognitrace-extension/`

## What to Exclude From Public Repo

- `cognitrace-demo/` (unless selected examples are intentionally curated)
- `Roadshow_Package_Final/`
- standalone commercialization docs at workspace root

## Migration Steps

1. Create a clean public repository.
2. Copy only approved public-scope files.
3. Re-run CI in the new repository.
4. Publish with a clear MVP capability statement.
